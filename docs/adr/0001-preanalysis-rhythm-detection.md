# ADR-0001: 预分析节奏检测方案

## 状态
已接受

## 背景
主视觉氛围和互动角色都需要与音乐节奏同步。有两种技术方案：
1. **实时检测**：播放时实时分析音频，检测节拍点
2. **预分析**：歌曲加载时预先分析，播放时直接使用

实时检测的优势是适应性强，但会消耗CPU资源，可能影响视觉渲染性能。

## 决策
采用**预分析方案**：
- 歌曲加载/缓冲时，提前分析BPM、节拍点、能量曲线等特征
- 将分析结果缓存（内存或本地存储）
- 播放时直接使用预分析数据驱动视觉和角色动作

## 理由
1. **性能优先**：避免播放时的实时计算，保证视觉渲染流畅
2. **延迟可控**：分析在加载阶段完成，不影响用户等待时间
3. **结果稳定**：同一首歌的节奏特征可复用，无需重复计算
4. **扩展性好**：未来可以将分析结果持久化，支持离线场景

## 后果
- **正面**：视觉同步流畅，CPU占用低，体验稳定
- **负面**：需要等待加载/分析完成才能开始播放（可通过预加载和缓存优化）
- **技术债**：需要设计缓存策略，避免内存占用过高
