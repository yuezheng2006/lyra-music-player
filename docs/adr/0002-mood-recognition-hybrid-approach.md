# ADR-0002: 情绪识别混合方案

## 状态
已接受

## 背景
情绪引擎需要识别歌曲情绪，用于驱动主视觉风格、角色动作强度和智能推荐。有几种数据来源：
- 第三方平台API标签（网易云、QQ音乐等）
- 本地音频特征分析（BPM、音调、能量等）
- 用户手动标注

单一方案的局限：
- 只用API标签：覆盖不全，依赖网络
- 只用本地分析：准确度不如人工标注
- 只靠用户标注：冷启动问题，用户负担重

## 决策
采用**混合方案**，按以下优先级：

1. **第一优先**：使用第三方音乐平台API的情绪标签
   - 从QQ音乐、网易云音乐等平台获取已有标签
   - 标签示例：快乐、悲伤、激昂、舒缓等

2. **第二优先**：本地音频特征分析
   - 当API无标签时，分析BPM（节奏快慢）、音调（大调/小调）、能量（振幅）
   - 通过规则或简单模型推断情绪

3. **用户修正 + 学习**：
   - 用户可以修正系统判断的情绪
   - 系统学习用户的修正习惯，优化后续推断

## 理由
1. **覆盖全面**：三层机制保证任何歌曲都能有情绪数据
2. **准确度优先**：优先使用人工标注（API标签），其次才是算法推断
3. **持续优化**：通过用户反馈不断提升准确度
4. **离线可用**：即使无网络，也能通过本地分析工作

## 实现细节
- API标签在歌曲元数据加载时获取（与播放历史聚合同步）
- 本地分析在预分析阶段完成（ADR-0001）
- 用户修正记录本地存储，与Play History关联

## 后果
- **正面**：情绪数据覆盖全面，准确度高，支持离线
- **负面**：需要维护多个数据源的逻辑，复杂度增加
- **技术债**：本地分析模型需要调优，避免误判

## 相关决策
- 与预分析节奏检测（ADR-0001）在同一阶段处理
- 为智能推荐系统（未来ADR）提供情绪数据基础
