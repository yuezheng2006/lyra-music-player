Stage API

Quickstart

Connection

1. Enable Stage Mode

在 Folia 设置中开启 Stage Mode，并复制 Bearer token。

2. Pick an endpoint

左侧目录按接口分组，适合先看文档再发请求。

3. Validate with live preview

所有请求都能先看到最终 method、headers 和 body，再真正发送。

Overview

Endpoint Surface

GET /stage/health

Public health check.

GET /stage/status

Read active stage state.

POST /stage/lyrics

Push parser-compatible lyric source.

POST /stage/session

Push media session by URL or file.

POST /stage/player/search

Search Netease songs.

POST /stage/player/play

Trigger Folia playback.

GET /stage/player/status

Read player playback state.

GET /stage/player/time

Read precise playback clock.

POST /stage/player/control

Send transport controls.

GET /stage/player/queue

Read player queue.

POST /stage/player/queue

Edit player queue.

WS /stage/player/ws

Subscribe to player events.

DELETE /stage/state

Clear current stage input.

Endpoint

                            
GETPOST/stage/player/queue

                        

                        

                            
                            
                        

                    

                    

                        

                            
读取或编辑正常播放器队列。GET 默认返回 100 条队列窗口，支持 offset、limit 和 around=current。

                            

                                
AuthBearer token

                                
Content-Typeapplication/json for POST

                            

                            
POST action

                            

                                
append / insert-next
需要 `songId` 或 `songIds`。（注：Folia 会对歌曲自动去重，若歌曲已在队列中则会被直接移动到目标位置，不会重复添加；请求本身如果有重复歌曲也会被合并。响应的 `deduplicated` 字段会体现去重状态。）

                                
remove / select
需要 `queueItemId` 或 `index`，`select` 会切到该队列项播放。使用 `queueItemId` 时会严格校验项匹配度。

                                
move
需要 `fromQueueItemId` 或 `fromIndex`，并提供 `toIndex`。使用 `queueItemId` 时会严格校验项匹配度。

                                
clear
清空队列中非当前播放项。
