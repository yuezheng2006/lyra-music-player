## 🚨 风险预案

### 风险 1: Vercel 部署失败
**应对：**
- 使用备用平台（Netlify, Railway）
- 提前准备构建日志，快速定位问题
- 如果 API 有问题，使用公共 API 临时替代

### 风险 2: 视频录制不理想
**应对：**
- 提前准备备用素材
- 简化场景，专注核心功能
- 使用屏幕录制 + 后期字幕说明

### 风险 3: 时间不够
**优先级排序：**
1. **必须完成**：部署 + 视频 + 核心功能
2. **重要但可省略**：UI 优化全部完成
3. **锦上添花**：展示页面、桌面端打包

### 风险 4: Bug 发现
**策略：**
- 优先修复影响核心功能的 Bug
- 视觉小问题可延后
- 准备 Bug 说明（作为改进方向）

## 💡 效率提升技巧

### 1. 并行工作
- 一人负责 UI 优化，一人负责部署配置
- 录制视频时可以同时完善文档

### 2. 使用 AI 工具
- ChatGPT/Claude：生成配音文案、文档内容
- ElevenLabs：AI 配音，节省时间
- Midjourney/Stable Diffusion：生成封面图（可选）

### 3. 复用模板
- 使用提供的 `showcase.html` 模板
- 使用 `VIDEO_SCRIPT.md` 脚本
- 使用 `vercel.json` 配置

### 4. 自动化脚本
```bash
# 快速部署脚本
#!/bin/bash
npm run build && vercel --prod

# 批量替换名称脚本
#!/bin/bash
find . -type f -name "*.tsx" -o -name "*.ts" | xargs sed -i '' 's/Auralis/Lyra/g'
```
